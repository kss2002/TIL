# Supabase 액세스 제어 (Access Control)

Supabase는 조직과 프로젝트 전반의 권한을 관리하기 위해 세분화된 액세스 제어 기능을 제공합니다.

[공식 문서](https://supabase.com/docs/guides/platform/access-control)

## 1. 역할(Roles) 정의

각 조직 및 프로젝트의 멤버는 다음 역할 중 하나를 가질 수 있습니다.

| 역할                       | 설명                                                                                                                   |
| :------------------------- | :--------------------------------------------------------------------------------------------------------------------- |
| **Owner (소유주)**         | 조직 및 프로젝트 리소스의 모든 것에 대한 전체 액세스 권한을 가집니다.                                                  |
| **Administrator (관리자)** | 조직 설정 업데이트, 프로젝트 이전, 새로운 소유주 추가를 **제외한** 모든 리소스에 대한 전체 액세스 권한을 가집니다.     |
| **Developer (개발자)**     | 조직 리소스에 대한 읽기 전용 권한과 프로젝트 리소스의 콘텐츠 접근 권한을 가지지만, 프로젝트 설정은 변경할 수 없습니다. |
| **Read-Only (읽기 전용)**  | 조직 및 프로젝트 리소스에 대한 읽기 전용 권한만 가집니다. (Team 및 Enterprise 플랜 전용)                               |

- 계정 생성 시 기본 조직이 생성되며 자동으로 **Owner**로 지정됩니다. 직접 생성하는 모든 조직에서도 **Owner** 역할을 부여받습니다.

---

## 2. 조직 멤버 관리

### 초대 및 관리

- 조직의 [팀 설정 페이지](https://supabase.com/dashboard/org/_/team)에서 이메일로 초대 링크를 보낼 수 있습니다. (유효 기간 24시간)
- **SAML SSO** 계정에서 보낸 초대는 동일한 ID 공급자의 다른 SAML SSO 계정으로만 수락할 수 있습니다.

### Management API를 이용한 멤버 조회

```bash
# 액세스 토큰 및 조직 ID 설정
export SUPABASE_ACCESS_TOKEN="your-access-token"
export ORG_ID="your-organization-id"

# 조직 멤버 목록 조회
curl "https://api.supabase.com/v1/organizations/$ORG_ID/members" \
  -H "Authorization: Bearer $SUPABASE_ACCESS_TOKEN"
```

### 소유권 이전

- 각 조직에는 최소 한 명의 소유주가 있어야 합니다.
- 본인이 유일한 소유주라면, 다른 사용자를 **Owner**로 초대하여 수락받거나 기존 멤버를 **Owner**로 승격시킨 후에만 조직을 떠날 수 있습니다.

## 3. 조직 범위 vs 프로젝트 범위 역할

- **조직 범위(Organization scoped)**: 멤버가 조직 수준의 역할을 가지면, 해당 조직 내의 모든 현재 및 미래 프로젝트에 대해 해당 권한을 가집니다.
- **프로젝트 범위(Project scoped)**: 특정 프로젝트에만 역할이 할당됩니다. 할당되지 않은 다른 프로젝트는 대시보드에서 보이지도 않으며 접근할 수 없습니다. (Team 및 Enterprise 플랜 전용)

## 4. 역할별 권한 상세 (Organization)

| 리소스                    | 작업                      | Owner | Administrator | Developer | Read-Only¹ |
| :------------------------ | :------------------------ | :---: | :-----------: | :-------: | :--------: |
| **조직 관리**             | 업데이트/삭제             |  ✅   |               |           |            |
| **OpenAI 텔레메트리**     | 업데이트                  |  ✅   |               |           |            |
| **멤버 관리**             | 목록 조회                 |  ✅   |      ✅       |    ✅     |     ✅     |
|                           | Owner 추가/제거           |  ✅   |               |           |            |
|                           | Admin/Developer 추가/제거 |  ✅   |      ✅       |           |            |
| **결제(Billing)**         | 송장/사용량 조회          |  ✅   |      ✅       |    ✅     |     ✅     |
|                           | 구독/결제 수단 업데이트   |  ✅   |      ✅       |           |            |
| **통합/OAuth 앱**         | 생성/업데이트/삭제        |  ✅   |      ✅       |           |            |
| **감사 로그(Audit Logs)** | 조회                      |  ✅   |      ✅       |    ✅     |     ✅     |

## 5. 역할별 권한 상세 (Project)

| 리소스           | 작업                     | Owner | Admin | Developer | Read-Only⁴ |
| :--------------- | :----------------------- | :---: | :---: | :-------: | :--------: |
| **프로젝트**     | 이전/삭제                |  ✅   |       |           |            |
|                  | 생성/업데이트/일시중지   |  ✅   |  ✅   |           |            |
| **데이터베이스** | 데이터 조회              |  ✅   |  ✅   |    ✅     |     ✅     |
|                  | 데이터 관리(쓰기)        |  ✅   |  ✅   |    ✅     |            |
| **인프라**       | 복제본/애드온 관리       |  ✅   |  ✅   |           |            |
| **API 설정**     | API 키/JWT 비밀번호 조회 |  ✅   |  ✅   |    ✅     |     ✅     |
|                  | 키 생성/설정 업데이트    |  ✅   |  ✅   |           |            |
| **인증(Auth)**   | 사용자 생성/삭제/관리    |  ✅   |  ✅   |    ✅     |            |
|                  | 설정/템플릿 업데이트     |  ✅   |  ✅   |           |            |
| **스토리지**     | 버킷 생성/수정/삭제      |  ✅   |  ✅   |    ✅     |            |
|                  | 파일 업로드/삭제         |  ✅   |  ✅   |    ✅     |            |
| **SQL 에디터**   | 쿼리 실행                |  ✅   |  ✅   |    ✅     |    ✅⁷     |
| **백업**         | 다운로드/복구            |  ✅   |  ✅   |    ✅     |            |

---

### 주석 (Footnotes)

1. **¹** Team 및 Enterprise 플랜에서만 사용 가능합니다.
2. **⁴** Team 및 Enterprise 플랜에서만 사용 가능합니다.
3. **⁵** Read-Only 역할도 시크릿(Secrets)에 접근할 수 있습니다.
4. **⁷** SELECT 쿼리 실행으로 제한됩니다. Read-Only 역할은 `supabase_read_only_user`를 사용하며, 이는 Postgres의 `pg_read_all_data` 역할을 가집니다.
