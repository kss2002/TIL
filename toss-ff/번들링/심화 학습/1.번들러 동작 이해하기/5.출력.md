## 출력(Output)

출력(Output)은 번들링된 결과물을 어디에 저장하고, 어떤 이름으로 만들지 결정하는 설정이에요. 웹팩 설정 파일의 `output` 필드에서 이를 정의할 수 있어요.

## 기본 출력 설정

가장 기본적인 출력 설정은 다음과 같아요.

```js
// webpack.config.js
const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
};
```

이 설정을 사용하면 웹팩은 `dist` 폴더에 `bundle.js`라는 이름으로 번들 파일을 생성해요.

## 동적 파일명 생성

실제 프로젝트에서는 파일명을 고정하지 않고 동적으로 생성하는 경우가 많아요. 이를 위해 플레이스홀더(placeholder)를 사용할 수 있어요.

### 주요 플레이스홀더

- `[name]`: 진입점 이름을 사용해요
- `[contenthash]`: 파일 내용을 기반으로 한 해시값을 추가해요
- `[chunkhash]`: 청크별로 고유한 해시값을 생성해요
- `[id]`: 번들의 고유 ID를 사용해요

```js
module.exports = {
  entry: {
    main: './src/index.js',
    vendor: './src/vendor.js',
  },
  output: {
    filename: '[name].[contenthash].js',
    path: path.resolve(__dirname, 'dist'),
  },
};
```

이렇게 설정하면 다음과 같은 파일들이 생성돼요:

- `main.abc123def.js`
- `vendor.xyz789uvw.js`

## Clean 옵션

매번 빌드할 때마다 기존 빌드 결과물을 삭제하고 싶다면 `clean` 옵션을 사용할 수 있어요.

```js
module.exports = {
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
    clean: true, // 빌드 전에 dist 폴더를 정리해요
  },
};
```

## Public Path 설정

배포된 파일들이 실제로 서빙될 때의 기본 경로를 설정할 수 있어요.

```js
module.exports = {
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
    publicPath: '/assets/', // CDN 또는 특정 경로에서 서빙할 때 유용해요
  },
};
```

이 설정을 사용하면 HTML에서 스크립트가 다음과 같이 참조돼요:

```html
<script src="/assets/bundle.js"></script>
```

## 라이브러리 번들 출력

라이브러리를 만들 때는 다른 프로젝트에서 사용할 수 있도록 특별한 설정이 필요해요.

```js
module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'my-library.js',
    path: path.resolve(__dirname, 'dist'),
    library: {
      name: 'MyLibrary',
      type: 'umd', // UMD 형식으로 내보내요
    },
    globalObject: 'this',
  },
};
```
